آزمایشگاه ممیزی تحلیل بازار یک سامانهٔ پژوهشی، محلی و مبتنی بر Docker است که رونویسی‌های مجاز و زمان‌دارِ تحلیل بازار را همراه با داده‌های بازار به یک بستهٔ شواهد قابل‌بازبینی تبدیل می‌کند. هوش مصنوعی در ساختاردهی ادعاها و اجرای قواعد صریح کمک می‌کند، اما خروجی آن هرگز به‌عنوان شاهد منبع پذیرفته نمی‌شود.

این سامانه ادعاهای تعریف‌شده را ارزیابی می‌کند. **توصیهٔ سرمایه‌گذاری ارائه نمی‌کند، سودآوری را اثبات نمی‌کند، صلاحیت هیچ تحلیل‌گری را تأیید نمی‌کند و عملکرد آینده را پیش‌بینی نمی‌کند.**

> [!IMPORTANT]
> **موتور عمومی و نمونه‌های ممیزی جدا از یکدیگرند:** این مخزن فقط موتور عمومی و بازاستفاده‌پذیر ممیزی و داده‌های ساختگیِ آزمایشی را در بر دارد. وب‌سایت برندشدهٔ هیچ ممیزی، شواهد مربوط به شخص مشخص، ویدئو یا رونویسی منبع، تصویر یا نشان تجاری، ردیف‌های دارای مجوز دادهٔ بازار، نتیجهٔ واقعی ممیزی و تنظیمات خصوصی دامنه، tunnel، اعتبارنامه و استقرار نمونهٔ واقعی در این مخزن قرار ندارد. هر ممیزی واقعی یک نمونهٔ جدا و تحت کنترل اپراتور است و مسئولیت حقوق منبع، حریم خصوصی، افشای رابطه و انتشار آن بر عهدهٔ همان اپراتور خواهد بود.

> [!CAUTION]
> فقط محتوایی را پردازش کنید که مالک آن هستید، برای استفاده از آن مجوز دارید یا قانوناً حق استفاده از آن را دارید. عمومی بودن یک نشانی اینترنتی به‌معنای اعطای مجوز نیست. تنها پس از بررسی شرایط پلتفرم، مجوزها، رضایت‌ها و قوانین قابل‌اجرا، مقدار `SOURCE_RIGHTS_ACKNOWLEDGED=true` را تنظیم کنید.

## هدف پروژه

تحلیل بازار معمولاً چند مسیر مشروط را ترسیم می‌کند و یک پیش‌بینی واحد نیست. ممیزی منصفانه باید شرط، محرک، ابطال، افق زمانی و موارد حذف‌شده را حفظ کند. زنجیرهٔ قابل‌بازبینی پروژه چنین است:

</div>

```text
گردآوری مجاز منبع
  → اعتبارسنجی کانال و بازهٔ زمانی
  → رونویسی مرجع + فهرست SHA-256
  → استخراج ساختاریافتهٔ ادعا
  → پیامدهای بازار هم‌تراز با زمان
  → حذف‌های قطعی + امتیازدهی محافظه‌کارانه
  → دادهٔ داشبورد + بستهٔ نهاییِ قابل‌راستی‌آزمایی
```

<div dir="rtl" lang="fa">

هر نتیجه به محدودهٔ خطوط منبع، هش رونویسی، دادهٔ بازار، پنجرهٔ ارزیابی، نسخهٔ پرامپت و طرح‌واره و تصمیم امتیازدهی متصل است. درصد نهایی، میزان هم‌راستایی سناریو و پیامد در میان ادعاهای فعال‌شده، قابل‌سنجش و شمرده‌شده است؛ نه «نرخ برد معاملاتی».

## شروع امن در پنج دقیقه

نمونهٔ همراه پروژه کاملاً ساختگی است: اجرای آن هیچ فراخوانی به ارائه‌دهنده یا API انجام نمی‌دهد و به کلید API نیاز ندارد. بااین‌حال، ساخت تازه و بدون Cache تصویر Docker برای دریافت تصویر پایه و وابستگی‌های پین‌شده به اینترنت نیاز دارد. تمام هویت‌ها، رونویسی‌ها، قیمت‌ها، ادعاها و نتایج نمونه تخیلی هستند.

</div>

```bash
umask 077 && cp .env.example .env
docker compose build
docker compose run --rm audit-lab python -m audit_lab.cli demo
docker compose run --rm audit-lab python scripts/generate_audit_report.py --workspace /workspace
docker compose run --rm audit-lab python -m audit_lab.cli finalize --synthetic-demo --workspace /workspace
docker compose run --rm audit-lab python -m audit_lab.cli verify-final --synthetic-demo --workspace /workspace
```

<div dir="rtl" lang="fa">

اگر Makefile در دسترس است، دستورهای معادل عبارت‌اند از:

</div>

```bash
make setup
make build
make demo
```

<div dir="rtl" lang="fa">

دستور `make demo` فایل PDF دوزبانه با ترتیب فارسی‌اول را نیز می‌سازد، بستهٔ نهایی را دوباره ایجاد می‌کند و یکپارچگی آن را می‌سنجد.

نمونه در پوشهٔ محلی `workspace/` نوشته می‌شود. پیش از تنظیم یک منبع واقعی، فایل `workspace/final_audit_summary.json` و فهرست‌های هش را بررسی کنید.

## اجرای ممیزی واقعی

۱. [حقوق و مجوز منبع](docs/fa/legal-and-rights.md)، [حریم خصوصی](docs/fa/privacy.md) و [روش‌شناسی](docs/fa/methodology.md) را بخوانید.

۲. فایل `.env.example` را به `.env` کپی کنید و نام موضوع، کانال، بازهٔ زمانیِ شامل دو سر، زبان زیرنویس، دامنهٔ ممیزی، منابع داده، `AUDIT_RELATIONSHIP_DISCLOSURE` و `CORRECTION_CONTACT` را تنظیم کنید.

۳. مجوز استفاده از منبع و پذیرش هزینهٔ API را جداگانه تأیید کنید. این تأییدها دروازهٔ ایمنی هستند و جای مجوز یا بررسی حقوقی را نمی‌گیرند.

۴. ابتدا فقط یک ویدئو را استخراج و بازبینی کنید؛ سپس برای اجرای کامل هزینه کنید.

۵. بستهٔ گردآوری و بستهٔ نهایی را راستی‌آزمایی کنید و پیش از انتشار، همهٔ نتیجه‌گیری‌ها را انسان بازبینی کند.

</div>

```bash
make doctor
make smoke
make collect
make manifest
make verify
make extract-claims ARGS="--limit 1"
make fetch-outcomes
make score ARGS="--limit 1"
make report
make pdf
make finalize
make verify-final
```

<div dir="rtl" lang="fa">

برای توضیح کامل مراحل، [راهنمای اپراتور](docs/fa/workflow.md) را ببینید. صرفاً به‌دلیل عمومی بودن یک محتوا، دستور `collect` را روی محتوای شخص ثالث اجرا نکنید.

## ورودی‌ها و منابع داده

- **شواهد منبع:** گردآورنده می‌تواند از زیرنویس‌های تنظیم‌شده استفاده کند و فراداده را ثبت کند؛ مسئولیت حق دسترسی و استفاده با اپراتور است. پروژهٔ عمومی هیچ نمونهٔ واقعی ممیزی را همراه خود منتشر نمی‌کند.
- **هوش مصنوعی:** استخراج ادعا و امتیازدهی تفسیری فقط در حالت `AUDIT_MODE=api` از OpenAI Responses API استفاده می‌کنند. رونویسی و شواهد ساختاریافتهٔ بازار به سرویس تنظیم‌شده ارسال می‌شود؛ [جریان داده و هزینهٔ OpenAI](docs/fa/openai-models-and-cost.md) را بخوانید.
- **پیامد رمزارز:** جفت‌های پشتیبانی‌شده از آرشیو عمومی یا API بازار نقدی Binance استفاده می‌کنند و در صورت وجود، منشأ و کنترل هش ثبت می‌شود.
- **بازارهای بین‌المللی:** برای دادهٔ دقیق و دارای مجوز، `csv` مسیر ترجیحی است. `yfinance` یک رابط غیررسمی برای Yahoo و مناسب نمونه‌سازی است؛ ممکن است از نماد جایگزین ساعتی استفاده کند و خوراک تضمین‌شده یا هم‌سطح بورس نیست.

## مرزهای اعتماد

- مدل محدودهٔ خطوط را انتخاب می‌کند؛ برنامه متن مرجع را مستقیماً از رونویسی هش‌شده کپی می‌کند.
- دارایی پشتیبانی‌نشده، پنجرهٔ ناقص، شرط فعال‌نشده، ادعای مبهم و زمینهٔ خارج از دامنه، «برد» محسوب نمی‌شود.
- حافظهٔ نهان هوش مصنوعی با اثرانگشت شواهد، پرامپت، سیاست، مدل و طرح‌واره کلیدگذاری می‌شود.
- راستی‌آزمایی هش ثابت می‌کند یک فایل موجود تغییر نکرده است؛ تضمین نمی‌کند اجرای دوبارهٔ API همان متن یا قضاوت را تولید کند.
- رونویسی ممکن است شامل متن تزریق پرامپت باشد؛ رونویسی شاهدِ غیرقابل‌اعتماد است، نه دستور. انتشار نتیجه به بازبینی انسانی نیاز دارد.
