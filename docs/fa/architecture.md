# معماری

[فارسی](architecture.md) · [English](../en/architecture.md) · [مستندات](index.md)

پروژه یک خط لولهٔ فایل‌محور است؛ فایل، هش، طرح‌واره و تنظیمات قرارداد اصلی‌اند و داشبورد فقط مصرف‌کننده است.

</div>

```text
منبع → شواهد خام → Manifest و رونویسی مرجع
                           ↓
OpenAI استخراج → ادعا و Cache
                           ↓
منبع بازار → سری عادی‌شده → پنجرهٔ پیامد
                           ↓
حذف قطعی → امتیاز تفسیری → تجمیع
                           ↓
گزارش → بستهٔ نهایی → Verify هش
```

<div dir="rtl" lang="fa">

`settings` اعتبارسنجی و Gate، `collect` مرز منبع، `manifest` دفتر ورود/حذف، `extract_claims` خروجی ساختاریافته، `fetch_outcomes` منبع بازار، `score_claims` مخرج و قواعد، `report` ارائه و `finalize/verify` بستهٔ قابل‌انتقال را مدیریت می‌کنند.

ورودی شبکه، متن، CSV و مدل غیرقابل‌اعتمادند. Workspace ممکن است حساس باشد. OpenAI در حالت API گیرندهٔ بیرونی است و Docker با Mount یا Port نادرست مرز محرمانگی نیست. Extension جدید باید Adapter باریک، منشأ کامل، آزمون ساختگی، نسخهٔ سیاست و مستندات دو‌زبانه داشته باشد.

</div>
