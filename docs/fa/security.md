# امنیت

[فارسی](security.md) · [English](../en/security.md) · [مستندات](index.md)

[سیاست اصلی امنیت](../../SECURITY.fa.md) را بخوانید و آسیب‌پذیری را خصوصی گزارش کنید.

رونویسی، فراداده، CSV، URL، آرشیو، پاسخ Provider/مدل و HTML را تا اعتبارسنجی مخرب فرض کنید. تزریق پرامپت، XSS، SSRF، پیمایش آرشیو، فرمول CSV، فایل بزرگ، مصرف منابع، Secret در Log و API بدون احراز هویت را پوشش دهید.

اجرای پیش‌فرض محلی است. انتشار به TLS، احراز هویت و مجوز، محدودیت نرخ/اندازه، Header امن، Escape، Mount و Egress محدود، Container غیر `root`، Backup، Log پاک، Pin وابستگی، SBOM و اسکن نیاز دارد. اعتبارنامه را در Git، لایهٔ Docker، History فرمان، گزارش، Screenshot یا Issue نگذارید.

دسترسی عمومی در مرز اثر نیز به‌صورت امن بسته می‌شود. پس از بازبینی شواهد، `review publication-accept` باید داشبورد، PDF و دفتر ادعای صریحاً فعال‌شده‌ای را که بررسی شده‌اند به هش متصل کند. `finalize` این ایست بازرسی فعلی را الزامی می‌کند و باید پیش از فعال‌سازی اتمیک Manifest انتشارِ متصل به هش، پوشه و ZIP نهایی کامل را راستی‌آزمایی کند. داشبورد و دانلودها باید در هر درخواست با هش‌های فعال‌شده منطبق باشند. جایگزینی یک اثر، اجرای دوبارهٔ `report`، بازتولید PDF یا تغییر هر یک از دفترهای بازبینی، دسترسی را تا تکرار پذیرش انتشار و موفقیت دوبارهٔ `finalize` می‌بندد.
