# انصاف، اصلاحات و حق پاسخ

[فارسی](fairness-and-publication.md) · [English](../en/fairness-and-publication.md) · [مستندات](index.md)

## پیش از تحلیل

موضوع یا کانال، بازهٔ شامل هر دو سر، دامنهٔ دسته‌ها، قواعد ورود، ارائه‌دهندگان، نمادهای جایگزین، سیاست امتیازدهی، زمان بلوغ پنجره و تعارض‌های شناخته‌شده را ثابت کنید. در `AUDIT_RELATIONSHIP_DISCLOSURE` اعلام کنید کار مستقل، سفارشی، دارای حمایت مالی، تبلیغاتی یا انجام‌شده برای خود موضوع است. پس از دیدن نتیجه، دامنه را تغییر ندهید مگر آن‌که هر دو نسخه و دلیل تغییر منتشر شوند.

## پیش از انتشار

- هش مجموعه و بستهٔ نهایی را راستی‌آزمایی کنید.
- حذف‌ها، دسته‌های ناشناخته، شرط‌های فعال‌شدن، ابطال‌ها، ترتیب مسیر چندمرحله‌ای، نتیجه‌های حساس به نماد جایگزین، محاسبهٔ مخرج، ترجمه‌ها و گزیده‌ها را بررسی کنید.
- بازبین انسانی و نسخهٔ روش‌شناسی، مدل و ارائه‌دهنده را مشخص کنید.
- یافتهٔ مبتنی بر شواهد را از نظر تبلیغاتی جدا کنید.
- شفاف‌سازی رابطه را با مهم‌ترین واقعیت آغاز کنید؛ برای نمونه سفارشی، خانوادگی، دارای حمایت مالی، تبلیغاتی یا انجام‌شده برای خود موضوع. آن را کنار تیتر یا پاسخ مستقیم و در صفحهٔ خلاصهٔ نخست PDF نشان دهید و متن کامل همراه با مسیر اصلاح را در بخش جزئیات نگه دارید.
- روشن کنید که ممیزی، سود، پیامد واقعی مخاطب، عملکرد آینده یا صلاحیت کلی تحلیل‌گر را ثابت نمی‌کند.
- کمترین گزیدهٔ ضروری از منبع را استفاده و حقوق دارایی را تأیید کنید.
- به شخص نام‌برده فرصت معقول بدهید خطای واقعی منبع را بررسی و پاسخ خود را ارائه کند، بی‌آن‌که کنترل روش‌شناسی را در اختیار بگیرد.

## ایست بازبینی انسانی متصل به هش

`PUBLICATION_MODE=public` فقط یک دروازهٔ فنی است، نه نتیجه‌گیری دربارهٔ انصاف. پس از بازبینی شواهد و نحوهٔ ارائه، یک انسان نام‌دار باید پذیرش صریح ثبت کند:

</div>

```bash
docker compose run --rm audit-lab python -m audit_lab.cli review status
docker compose run --rm audit-lab python -m audit_lab.cli review accept \
  --reviewer "نام بازبین" --notes "دامنه، گزیده‌ها، ترجمه، مخرج و محدودیت‌ها بررسی شد"
docker compose run --rm audit-lab python -m audit_lab.cli report
docker compose run --rm audit-lab python scripts/generate_audit_report.py --workspace /workspace
# پیش از ادامه، dashboard_data.json، فایل PDF و دفتر عمومی اختیاری را بررسی کنید.
docker compose run --rm audit-lab python -m audit_lab.cli review publication-accept \
  --reviewer "نام بازبین" --notes "داشبورد، PDF و دفتر عمومی اختیاری بررسی شد"
docker compose run --rm audit-lab python -m audit_lab.cli finalize
docker compose run --rm audit-lab python -m audit_lab.cli verify-final
```

<div dir="rtl" lang="fa">

دفتر بازبینیِ فقط‌افزودنی، رویدادها را با زنجیرهٔ هش ثبت و پذیرش را به حالت عمومی، مانیفست مجموعه، دفتر ادعا، تصویر ثابت یا فایل پیامد، اجرای امتیازدهی و دفتر امتیاز متصل می‌کند. تغییر حالت یا هر فایل متصل، پذیرش را قدیمی می‌کند. برای پس‌گرفتن مجوز از `review revoke` استفاده کنید و دفتر را دستی ویرایش نکنید. نام و یادداشت بازبین در بستهٔ خصوصی می‌مانند و DTO عمومی داشبورد آن‌ها را برنمی‌گرداند.

پس از ایست بازبینی شواهد، داشبورد، PDF و هر دفتر عمومی ادعا که صریحاً فعال شده است را دوباره بسازید و بررسی کنید. `review publication-accept` ایست دوم را برای بایت‌های دقیق همین فایل‌های ارائه ثبت می‌کند. در حالت عمومی، `finalize` هر دو ایست را الزامی می‌کند، پوشه و ZIP نهایی را راستی‌آزمایی می‌کند و فقط سپس مانیفست انتشار را فعال می‌کند. داشبورد و دانلودهای عمومی فقط تا زمانی ارائه می‌شوند که بایت‌های فعلی با آن مانیفست مطابق باشند. جایگزینی یا حذف فایل منتشرشده باعث بسته‌شدن امن دسترسی می‌شود. پس از اجرای دوبارهٔ `report`، بازتولید PDF یا تغییر هر دفتر بازبینی، پیش از ازسرگیری انتشار باید `review publication-accept` و سپس `finalize` دوباره با موفقیت اجرا شوند.

## اصلاحات

مقدار اعتبارسنجی‌شدهٔ `CORRECTION_CONTACT` را کنار شناسهٔ تغییرناپذیر ممیزی، تاریخ، نسخه و changelog منتشر کنید. اصلاح را در یکی از دسته‌های منبع/فراداده، رونویسی، دادهٔ بازار، استخراج ادعا، امتیازدهی، ترجمه یا ارائه طبقه‌بندی کنید. بستهٔ منتشرشده را پنهانی جایگزین نکنید؛ نسخهٔ تازه صادر کنید، هش قبلی را نگه دارید، اثر تغییر بر ادعاها، مخرج و تیتر را توضیح دهید و نسخه‌ها را به هم پیوند دهید.

انتقاد یا پاسخ نیک‌نیت به‌طور خودکار امتیاز را تغییر نمی‌دهد؛ خطای شواهد تغییر می‌دهد. تلافی، آزار، دست‌کاری تعامل، حذف گزینشی و پنهان‌کردن رابطهٔ مالی ممنوع است.
