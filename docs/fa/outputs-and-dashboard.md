# خروجی و داشبورد

[فارسی](outputs-and-dashboard.md) · [English](../en/outputs-and-dashboard.md) · [مستندات](index.md)

مرحلهٔ `report` دادهٔ قطعی ارائه را از فایل‌های تحلیل نهایی مشتق می‌کند و نباید تصمیم ادعا یا امتیاز را تغییر دهد. داشبورد وب و PDF نماهایی از همان داده‌اند، نه مخزن مستقل شواهد.

## زمینهٔ الزامی ارائه

هر تیتر باید بازهٔ ممیزی، تعداد ویدئوهای کشف‌شده/واردشده/حذف‌شده، تعداد کل/استخراج‌شده/شمرده‌شدهٔ ادعاها، توزیع نتیجه، شمار شرط‌های فعال‌نشده و دادهٔ ناکافی، دسته‌های دامنه، ارائه‌دهنده و نماد جایگزین، نسخهٔ مدل و روش‌شناسی، محدودیت‌ها، مقدار `AUDIT_RELATIONSHIP_DISCLOSURE` و مقدار `CORRECTION_CONTACT` را نشان دهد.

مقدار تجمیعی را «هم‌راستایی سناریو و پیامد در ادعاهای شمرده‌شده، فعال و قابل‌راستی‌آزمایی» بنامید. مخرج را کنار درصد نمایش دهید. هرگز سود مخاطب، عملکرد معاملاتی، دقت آینده، گواهی یا تضمین را القا نکنید.

## جداسازی عمومی و خصوصی

نمای خصوصی ممکن است مسیر کامل شواهد و گزیده‌های طولانی‌تر را نشان دهد. نمای عمومی به‌طور پیش‌فرض فقط کمترین گزیدهٔ ضروری، متن escapeشده، بدون مسیر محلی، بدون دانلود رونویسی خام، بدون حافظهٔ نهان API و بدون endpoint حساسِ فاقد احراز هویت ارائه می‌کند. بندانگشتی، تصویر چهره، نشان تجاری و قلم باید حقوق ثبت‌شده در `ASSET_LICENSES.yml` داشته باشند.

تصویر JSON در `reports/dashboard_data.json` یک فایل داخلی ارائه است و ممکن است جزئیات خصوصی لازم برای PDF دوزبانه داشته باشد؛ این فایل قرارداد API عمومی نیست. صفحهٔ HTML و `/api/dashboard` از یک DTO تجمیعی و مبتنی بر فهرست مجاز استفاده می‌کنند. وقتی `PUBLIC_CLAIM_LEDGER=false` است، متن و گزیدهٔ ادعا، استدلال مدل، نشانی ویدئو، مسیر محلی یا خطای ارائه‌دهنده را برنمی‌گردانند. حالت خصوصی با «پیش‌نمایش خصوصی» و حالت عمومی بدون بازبینی معتبر با «در انتظار بازبینی انسانی» مشخص می‌شود. در هر دو حالت، دانلود گزارش و دفتر ادعا بسته است.

پس از پذیرش بازبینی شواهد، داشبورد و PDF بازتولیدشده و دفتر عمومی اختیاری را بررسی کنید و سپس `review publication-accept` را اجرا کنید. `finalize` عمومی این ایست ارائه را الزامی می‌کند، پوشه و ZIP نهایی را راستی‌آزمایی می‌کند و فقط پس از آن مانیفست انتشارِ متصل به فایل‌های پذیرفته‌شده را فعال می‌کند. داشبورد و هر دانلود، فایل قابل‌ارائه را با هش فعال‌شده مقایسه می‌کنند. فایل حذف‌شده، بازتولیدشده یا جایگزین‌شده به‌جای به‌ارث‌بردن پذیرش قبلی باعث بسته‌شدن امن می‌شود.

## خروجی دوزبانه و دسترس‌پذیر

تغییر زبان باید متن، جهت، قالب عدد و تاریخ در موارد لازم، برچسب، متن جایگزین و گزارش قابل‌دانلود را عوض کند، نه فقط ناوبری را. فارسی از چیدمان راست‌به‌چپ استفاده می‌کند، اما نمودار، هش، نماد، زمان و کد چپ‌به‌راست می‌مانند. کاهش حرکت، ناوبری صفحه‌کلید، کنتراست، عنوان‌های معنایی و برچسب‌های صفحه‌خوان را رعایت کنید.

نمونهٔ ساختگی فقط با حفظ اعلان ساختگی‌بودن می‌تواند عمومی نمایش داده شود.

## ساخت PDF دوزبانه

پس از مرحلهٔ `report`، سند فارسی‌اول و انگلیسی‌دوم را بسازید و بررسی کنید، فایل‌های دقیق ارائه را بپذیرید و سپس نهایی‌سازی را انجام دهید تا PDF وارد بستهٔ قابل‌راستی‌آزمایی با هش شود:

</div>

```bash
docker compose run --rm audit-lab python scripts/generate_audit_report.py --workspace /workspace
# پیش از ادامه، dashboard_data.json، فایل PDF و دفتر عمومی اختیاری را بررسی کنید.
docker compose run --rm audit-lab python -m audit_lab.cli review publication-accept \
  --reviewer "نام بازبین" --notes "داشبورد، PDF و دفتر عمومی اختیاری بررسی شد"
docker compose run --rm audit-lab python -m audit_lab.cli finalize
docker compose run --rm audit-lab python -m audit_lab.cli verify-final
```

<div dir="rtl" lang="fa">

خروجی پایدار در `workspace/reports/audit-report.pdf` نوشته می‌شود. این سند فقط نمایی از `dashboard_data.json` است و نتیجهٔ ممیزی را دوباره محاسبه یا بهتر نمی‌کند.

برای ممیزی عمومی واقعی، پذیرش فعلی شواهد را ثبت کنید، `report` را دوباره بسازید، PDF فارسی‌اول/انگلیسی‌دوم و هر دفتر عمومی اختیاری را تولید و بررسی کنید، `review publication-accept` را ثبت کنید و سپس بدون تغییر فایل پذیرفته‌شده، نهایی‌سازی و راستی‌آزمایی را انجام دهید. تغییر مجموعه، پیامد یا امتیاز، پذیرش شواهد را قدیمی می‌کند. اجرای دوبارهٔ `report`، بازتولید PDF، جایگزینی دفتر اختیاری یا تغییر هر دفتر بازبینی، مجوز انتشار و مانیفست فعال را نامعتبر می‌کند؛ پیش از ارائهٔ داشبورد یا دانلودها، `review publication-accept` و `finalize` را تکرار کنید.
