<div dir="rtl" lang="fa">

مجموعهٔ پیش‌فرض آزمون آفلاین و ساختگی است. آزمون‌های شبکه یا مدل پولی باید جداگانه علامت‌گذاری، با انتخاب صریح فعال و از نظر بودجه محدود شوند و هرگز برای pull request عادی الزامی نباشند.

اعتبارسنجی تنظیمات، مرز تاریخ، ناسازگاری کانال، منشأ زیرنویس، دفتر موارد تکراری و حذف‌شده، شاهد خطوط مرجع، خروجی خراب مدل، تزریق پرامپت، بی‌اعتبارشدن حافظهٔ نهان، شکاف/منطقهٔ زمانی/نماد جایگزین ارائه‌دهنده، ترتیب رویدادها، حذف‌های قطعی، محاسبهٔ مخرج، خرابی هش یا ZIP، پاک‌سازی داده، همسانی RTL و i18n، XSS و فایل مخرب را بیازمایید.

ارائه‌دهندهٔ جدید باید دادهٔ عادی‌سازی‌شده و دارای منشأ کامل برگرداند و از پاسخ ساختگی آزمون استفاده کند. دستهٔ جدید از تنظیمات می‌آید، نه از کد ثابت برای یک شخص نام‌برده. تغییر روش‌شناسی، پرامپت، طرح‌واره یا سیاست به افزایش نسخه، دادهٔ ثابت آزمون بازگشتی، یادداشت مهاجرت و مستندات هر دو زبان نیاز دارد.

پیش از بازبینی، کنترل قالب/سبک/نوع تنظیم‌شده در مخزن، آزمون‌های واحد، نمونهٔ ساختگی سرتاسری، کنترل پیوند و همسانی مستندات، اسکن کلیدهای محرمانه و ساخت کانتینر را اجرا کنید. نتیجهٔ مرجع آزمون را بدون توضیح دلیل معنایی تغییر ندهید.
