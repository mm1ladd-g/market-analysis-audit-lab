# مدل OpenAI، جریان داده و هزینه

[فارسی](openai-models-and-cost.md) · [English](../en/openai-models-and-cost.md) · [مستندات](index.md)

OpenAI برای Demo لازم نیست و فقط برای استخراج یا امتیاز تفسیریِ بدون Cache در حالت API استفاده می‌شود.

استخراج، فراداده، هش و رونویسی شماره‌گذاری‌شده را می‌فرستد. امتیازدهی، ادعا و پنجرهٔ ساختاریافتهٔ پیامد را ارسال می‌کند. درخواست `store=false` دارد، اما اپراتور باید مستندات جاری و شرایط حساب را برای پردازش و نگهداری بررسی کند؛ این گزینه به‌تنهایی وعدهٔ عدم نگهداری نیست.

شناسهٔ دقیق مدل موجود را تنظیم و مدل درخواستی و بازگشتی را ثبت کنید. مدل ممکن است بازنشسته شود. کار پولی به حالت API، کلید Runtime و تأیید هزینه نیاز دارد. با `--limit 1` آغاز، Concurrency را کم و قیمت رسمی را هر بار بررسی کنید. خطای ناموفق نیز ممکن است هزینه‌ای داشته باشد که Log محلی نمی‌بیند.

Cache به شواهد، پرامپت، طرح‌واره/سیاست، مدل و Reasoning وابسته است. `--force` هزینهٔ دوباره دارد. ورودی یکسان تضمین‌کنندهٔ متن یا قضاوت یکسان نیست؛ اثر پذیرفته‌شده را حفظ و هش کنید.
