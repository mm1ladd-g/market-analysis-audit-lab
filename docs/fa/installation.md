## مسیر پشتیبانی‌شده

Docker Compose مسیر بازتولیدپذیر پروژه است. از نسخهٔ جاری و پشتیبانی‌شدهٔ Docker در macOS، Linux یا Windows همراه WSL2 استفاده کنید. اجرای بدون دسترسی root در Linux ترجیح دارد. ساخت تازه برای دریافت تصویر پایه و وابستگی‌های ثابت‌شده به شبکه نیاز دارد؛ هنگام اجرا فقط مراحل گردآوری، مدل و بازار که اپراتور صریحاً تنظیم کرده است به شبکه دسترسی می‌خواهند.

</div>

```bash
git clone https://github.com/mm1ladd-g/market-analysis-audit-lab.git
cd market-analysis-audit-lab
umask 077 && cp .env.example .env
docker compose build
docker compose run --rm audit-lab python -m audit_lab.cli doctor
```

<div dir="rtl" lang="fa">

سرویس باید به‌طور پیش‌فرض فقط به localhost متصل باشد. صرفاً برای ساده‌ترکردن آزمون، رابط عمومی اضافه نکنید.

## برنامه‌ریزی ظرفیت

مصرف دیسک به حجم زیرنویس و فراداده، بندانگشتی‌های اختیاری، تفکیک زمانی بازار، حافظهٔ نهان API و بسته‌های نهایی بستگی دارد. کار را با چند گیگابایت فضای آزاد آغاز و `workspace/` را پایش کنید. بازه‌های طولانی و دادهٔ یک‌دقیقه‌ای رمزارز می‌توانند به‌سرعت بزرگ شوند. نیاز پردازنده معمولاً کم است، مگر آن‌که گفتاربه‌متن محلی اضافه شود؛ استخراج مبتنی بر API بیشتر به شبکه و بودجه وابسته است.

## Python محلی

توسعهٔ محلی با نسخهٔ Python اعلام‌شده در پروژه و فایل دقیق قفل وابستگی‌ها ممکن است، اما مسیر ترجیحی بازتولید نیست.

</div>

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install --require-hashes -r requirements-dev.lock
python -m unittest discover -s tests -v
```

<div dir="rtl" lang="fa">

هنگام بازتولید یک انتشار، وابستگی‌ها را بدون بازبینی ارتقا ندهید.

## ارتقا

`CHANGELOG.md` و یادداشت انتشار را بخوانید، از فضای کاری نسخهٔ پشتیبان بگیرید، tag دقیق را دریافت کنید، تصویر را بدون واردکردن کلیدهای محرمانه بسازید، آزمون‌ها و نمونهٔ ساختگی را اجرا کنید و پیش از اجرای دوبارهٔ مراحل پولی، یک بستهٔ موجود را راستی‌آزمایی کنید. طرح‌واره‌های پیش از نسخهٔ ۱٫۰ ممکن است به مهاجرت نیاز داشته باشند.

## حذف

توقف یا حذف کانتینر، شواهد موجود در پوشهٔ متصل‌شده را پاک نمی‌کند. تنها پس از رعایت سیاست نگهداری، فضای کاری، نسخه‌های پشتیبان، گزارش‌های خروجی و ذخیره‌سازی بیرونی را حذف کنید. اگر اطلاعات محرمانه افشا شده‌اند، اعتبارنامه‌ها را جداگانه تعویض کنید.
