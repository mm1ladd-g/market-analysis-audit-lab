# حقوق و مجوز منبع

[فارسی](legal-and-rights.md) · [English](../en/legal-and-rights.md) · [مستندات](index.md)

این راهنمای کاهش خطر است، نه مشاورهٔ حقوقی.

## دروازهٔ حقوق محتوا

فقط محتوایی را وارد کنید که مالک آن هستید، برای پردازش آن مجوز دارید یا قانوناً حق استفاده دارید. عمومی بودن URL مجوز نیست. پروژه اجازهٔ دانلود، رونویسی، ذخیره، ارسال به هوش مصنوعی، تغییر، بازنشر یا انتشار شخص ثالث نمی‌دهد.

شرایط YouTube دسترسی خودکار و دانلود را جز در صورت اجازهٔ سرویس، رضایت قبلی یا قانون محدود می‌کند. کلاینت‌های API نیز تعهدهای بیشتری دربارهٔ حریم خصوصی، انتساب، نگه‌داری، هویت بصری و منع جمع‌آوری خودکار داده دارند. نسخهٔ جاری [شرایط YouTube](https://www.youtube.com/static?template=terms) و [سیاست‌های توسعه‌دهندگان YouTube API](https://developers.google.com/youtube/terms/developer-policies) را بررسی کنید. مقدار `SOURCE_RIGHTS_ACKNOWLEDGED=true` فقط ثبت تأیید اپراتور است؛ مصونیت یا مجوز ایجاد نمی‌کند.

خروجی دریافت‌شده از مالک محتوا، زیرنویس ارائه‌شده از سوی دارندهٔ حق یا API مجاز را ترجیح دهید. برای ویدئوهای دلخواه اشخاص ثالث، دانلودکنندهٔ صوت پیش‌فرض طراحی نکنید. اگر به استثنای قانونی استناد می‌کنید، مشاورهٔ متناسب با حوزهٔ قضایی بگیرید و کپی یا انتشار را به حد ضروری محدود کنید.

## سایر حقوق

دادهٔ بازار، لوگو، تصویر چهره، تصویر بندانگشتی، قلم و علامت تجاری هرکدام شرایط جداگانه دارند. منشأ، مالک، تاریخ دریافت، مجوز یا اجازه، کاربردهای مجاز، شیوهٔ انتساب و تاریخ انقضا را ثبت کنید. دادهٔ خام تأمین‌کننده را بدون حق بازنشر منتشر نکنید. استفاده از نام یا نشان برای شناسایی، به معنای وابستگی یا تأیید نیست.

## خطر مالی و اعتباری

این نرم‌افزار زیرساخت پژوهشی است؛ نه توصیهٔ سرمایه‌گذاری، اثبات سودآوری، گواهی صلاحیت تحلیل‌گر یا پیش‌بینی عملکرد آینده. از نتیجه‌گیری افترا‌آمیز یا گمراه‌کننده پرهیز کنید، زمینه و عدم‌قطعیت را حفظ کنید، حمایت مالی را اعلام کنید، مسیر اصلاح و حق پاسخ فراهم کنید و انتشار را به تأیید انسانی برسانید.

مجوز Apache-2.0 پروژه فقط کد و مستندات اصلی را پوشش می‌دهد، نه شواهدی را که اپراتور وارد می‌کند.
